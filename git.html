Базові означення

A version control system (VCS) allows you to track the history of a collection of files.

In a distributed version control system each user has a complete local copy of a repository on his individual computer. The user can copy an existing repository. This copying process is typically called cloning and the resulting repository can be referred to as a clone.

Git is currently the most popular implementation of a distributed version control system.

Types of repositories:

bare target server do not allow the user to modify locally files
non-bare target user allow you to create new changes through modification of files

States of files in the working tree

  • untracked - file never staged nor committed
  • tracked - committesd and not staged
  • staged - staged to be included in the next commit
  • modified - file has changed and this change is not staged

Git supports branching which means that you can work on different versions of your collection of files.

Name Definition
Branch is a named pointer to a commit.
Commit When you commit your changes into a repository this creates a new commit object in the Git repository. This commit object uniquely identifies a new revision of the content of the repository.
HEAD is a symbolic reference most often pointing to the currently checked out branch.
Index Index is an alternative term for the staging area.
Repository contains the history, the different versions over time and all different branches and tags.
Revision a version of the source code.In Git - commit object.
Staging area is the place to store changes in the working tree before the commit.
Tag points to a commit which uniquely identifies a version of the Git repository. Branches move when a new commit is created while tags always point to the same commit.
Working tree contains the set of working files for the repository.

Основні команди

  • git init   #initialize Git repository in current location
  • git status   #get current project's state
  • git add .   #to add all files to staging area
  • git add '*.txt'
  • git commit -m "Some text"   #to store staged changes with some description
  • git log   #get history of committed changes
  • git remote add origin https://github.com/try-git/try_git.git  #add remote repository with name 'origin'
  • git push -u origin master  #add local committed changes in branch 'master' to remote repository 'origin'. -u - save parameters
  • git pull origin master  #get changes from remote repository
  • git diff HEAD  #see difference from last commit
  • git diff --staged  #see staged changes
  • git reset text.txt  #unstage file text.txt
  • git checkout -- cat.txt  #get rid of all the changes since the last commit for cat.txt
  • git branch diffStaff  #create branch with name 'diffStaff'
  • git checkout diffStaff  #switch to 'diffStaff' branch
  • git rm '*.txt'  #removes files and stages the removal
  • git merge diffStaff  #merge current branch and branch 'diffStaff'
  • git branch -d diffStaff  #delete branch
